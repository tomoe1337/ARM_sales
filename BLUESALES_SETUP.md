# Настройка обратной синхронизации с BlueSales

## Переменные окружения

Добавьте в `.env`:

```env
BLUESALES_LOGIN=your_login@example.com
BLUESALES_API_KEY=your_api_key_here
BLUESALES_SYNC_ENABLED=true
```

## Как работает

При создании/обновлении клиента в системе:
1. `ClientObserver` перехватывает событие
2. Проверяет, включена ли синхронизация (`BLUESALES_SYNC_ENABLED=true`)
3. Трансформирует данные клиента в формат BlueSales
4. Отправляет в BlueSales API
5. Обновляет `bluesales_id` и `bluesales_last_sync`

## Логи

Все операции логируются:
- Успешное создание/обновление
- Ошибки синхронизации (не прерывают работу основной системы)

## Отключение

Для временного отключения:
```env
BLUESALES_SYNC_ENABLED=false
```

## Примечания

- Синхронизация работает **только при наличии всех трех переменных**
- Ошибки синхронизации **не блокируют** создание/обновление клиентов в системе
- Позднее можно привязать креды к отделам продаж
